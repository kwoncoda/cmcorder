# design_fix 검수 지시서

## 검수 원칙

- 수동 QA는 현재 구현 화면을 `docs/design-bundle/` 원본과 직접 비교하는 방식으로 진행합니다.
- 새 디자인처럼 더 예쁘게 보이는지가 아니라, 원본 디자인과 얼마나 동일한지가 기준입니다.
- 기능 로직 회귀가 없어야 합니다.
- `.env`, 비밀키, DB 실데이터, 세션 파일은 열람하지 않습니다.

## 사전 준비

- 브랜치가 `design_fix`인지 확인합니다.
- 원본 비교 기준 파일을 열어 둡니다.
  - `docs/design-bundle/screens-admin.jsx`
  - `docs/design-bundle/screens-customer.jsx`
  - `docs/design-bundle/app.css`
  - `docs/design-bundle/tokens.css`
- 데스크톱 기준으로 본부 대시보드 1024px 이상 폭을 확인합니다.
- 모바일 기준으로 사용자 주문 화면 390px 전후 폭을 확인합니다.

## 필수 수동 QA 체크리스트

1. 영업 중 OPEN 초록 dot
   - 어드민 상단 오른쪽 상태 표시 옆에 초록 dot이 보이는지 확인합니다.
   - 본부 대시보드 `영업 중` 상태 바 왼쪽에도 초록 dot이 보이는지 확인합니다.
   - dot 크기, 위치, 여백이 `design_bundle`과 유사한지 확인합니다.

2. 장사 시작 전 화면
   - `CLOSED` 상태에서 장사 시작 카드가 `design_bundle`의 `.start-cta.urgent`와 거의 동일한지 확인합니다.
   - 노란색 강조, dashed border, 좌우 striped line, mascot box, 버튼 크기와 여백이 원본과 비슷해야 합니다.

3. 장사 시작 버튼 기능 연결
   - 장사 시작 버튼 클릭 시 실제 장사 시작 기능이 동작하는지 확인합니다.
   - 클릭 후 상태가 `OPEN`으로 바뀌고 사용자 주문 가능 상태로 전환되는지 확인합니다.

4. 장사 시작 전 6개 상태 박스
   - `CLOSED` 상태에서도 본부 대시보드 아래에 6개 상태 박스가 모두 보이는지 확인합니다.
   - `ORDERED`, `TRANSFER_REPORTED`, `PAID`, `COOKING`, `READY`, `HOLD` 컬럼이 사라지면 실패입니다.

5. 어드민 메뉴 가격 설정 저장/취소 이모지
   - 메뉴 가격 편집 진입 시 저장/취소가 작은 `✓`/`✕` 아이콘 버튼 형태로 보이는지 확인합니다.
   - 아이콘 버튼이 가격 셀 안에서 원본처럼 compact하게 보이는지 확인합니다.
   - 저장, 취소, Enter 저장, Escape 취소가 그대로 동작하는지 확인합니다.

6. CompletePage 한글 부 카피 제거
   - 주문 접수 완료 페이지에서 `치킨 디너 위너!` 문구가 사라졌는지 확인합니다.
   - `WINNER WINNER` / `CHICKEN DINNER!` 2줄 문구는 유지되는지 확인합니다.

7. CompletePage 상단 연출 박스
   - `WINNER WINNER CHICKEN DINNER` 위의 dogtag/네모 연출 박스가 `design_bundle`처럼 떨어지는 느낌과 pulse 느낌을 주는지 확인합니다.
   - 첫 진입 animation과 재진입 시 과도한 반복 방지 정책이 깨지지 않았는지 확인합니다.

8. 입금 안내 박스
   - 입금 안내 박스가 `design_bundle`과 동일한 색감/여백/구조인지 확인합니다.
   - 은행/예금주, 계좌번호, 금액, 계좌 복사, 금액 복사 버튼 배치가 원본과 유사해야 합니다.
   - 금액은 mono 큰 글씨와 노란 accent, dashed top border로 강조되어야 합니다.

9. 쿠폰 영역 문구
   - 쿠폰 영역에는 `컴모융 학생 1,000원 할인`만 보여야 합니다.
   - `학번 9자리 + 이름 입력 시 활성화됩니다.` 보조 안내 문구가 보이면 실패입니다.
   - 학번/이름 조건 충족 전후로 쿠폰 활성/비활성 동작은 유지되어야 합니다.

10. 본부 대시보드 주문 카드 가독성
    - 주문 카드가 너무 어둡지 않고 글자가 잘 보이는지 확인합니다.
    - 주문번호, 이름, 금액, 경과 시간, 버튼 글자가 배경과 충분히 구분되어야 합니다.

11. 본부 대시보드 원본 색감
    - 본부 대시보드의 색감, 노란색 포인트, 카드 UI가 `design_bundle`과 유사한지 확인합니다.
    - `--color-accent` 노란색 포인트가 원본처럼 쓰이는지 확인합니다.
    - 추가 문구/메뉴 내역 문구를 제외한 카드/박스/버튼/스크롤 느낌이 원본과 달라지면 실패입니다.

12. 취소/보류 버튼 톤
    - 취소 버튼이 너무 튀는 빨간색 배경으로 보이지 않는지 확인합니다.
    - 보류 버튼도 위험 의미는 유지하되 원본 카드 액션 버튼 톤과 어울리는지 확인합니다.
    - primary 액션은 노란색 강조로 구분되는지 확인합니다.

13. 모바일 디자인
    - 사용자 주문 접수 완료 페이지와 주문 정보 페이지를 390px 전후 모바일 폭에서 확인합니다.
    - 텍스트가 버튼/카드 밖으로 넘치거나 서로 겹치지 않아야 합니다.
    - sticky bar가 본문을 가리더라도 주요 CTA와 내용 흐름이 원본처럼 유지되어야 합니다.

14. 기존 기능 회귀 확인
    - 장사 시작이 동작하는지 확인합니다.
    - 주문 상태 이동이 동작하는지 확인합니다.
    - 쿠폰 사용이 동작하는지 확인합니다.
    - 이체 완료 요청이 동작하는지 확인합니다.
    - 관리자 상태 변경이 동작하는지 확인합니다.

## 추가 권장 QA

- `OPEN` 상태와 `CLOSED` 상태를 각각 확인합니다.
- 주문 카드가 비어 있는 컬럼과 주문이 있는 컬럼을 모두 확인합니다.
- `TRANSFER_REPORTED` 카드에서 `확인`/`보류` 버튼이 모두 보이고 동작하는지 확인합니다.
- `HOLD` 카드에서 `이체 확인`/`취소` 버튼이 모두 보이고 동작하는지 확인합니다.
- 가격 저장 후 메뉴 가격이 사용자 주문 화면에도 기존 정책대로 반영되는지 확인합니다.
- 쿠폰 사용 주문 완료 후 CompletePage receipt에 쿠폰 할인 표시가 기존처럼 유지되는지 확인합니다.

## 실패 기준

- `design_bundle`에 없는 새 색감/버튼/카드 스타일이 도입된 경우.
- 초록 OPEN dot이 안 보이거나 텍스트와 겹치는 경우.
- 장사 시작 전 6컬럼 상태 박스가 사라진 경우.
- 입금 안내 박스가 원본 구조와 크게 달라진 경우.
- 주문 카드 글자 대비가 낮아 운영자가 읽기 어려운 경우.
- 기능 로직, API, DB schema, 상태 전이 정책, 쿠폰 정책이 변경된 경우.
