# Task 4.3·4.4 — CartPage (C-2 인벤토리) + CheckoutPage (C-3 주문 정보 입력)

## 목표

- **Task 4.3**: 카트 화면 (G11 "인벤토리 (N개)" 라벨) + 수량 변경 + sticky CTA. 비어있을 때 EmptyState 마스코트 fallback. Zustand 셀렉터로 합계 즉시 반영.
- **Task 4.4**: 학번·이름·"학번 없음"·수령 방식·테이블·쿠폰 입력 폼. ADR-019 학번 정규식 `^\d{2}\d{2}37\d{3}$` (학과 코드 37 강제). ADR-021 외부인 모드 — 학번/쿠폰 숨김. ADR-020 Pattern B 페이로드 (menu_id·quantity·coupon만, 가격 자체 계산은 서버).

두 페이지 모두 **≤120줄** (§3.5 1조) 회귀 강제.

## 한 일

### CartPage.jsx (86줄)

- `useCartStore` **셀렉터 함수** 4개 직접 구독: `(s) => s.items`, `cartSelectors.totalQty`, `cartSelectors.totalPrice`, `(s) => s.changeQty`, `(s) => s.removeItem`. 전체 객체 destructure 금지 (§3.5 2조).
- 3분기 처리:
  - **Empty**: `items.length === 0` → `<EmptyState variant="page" mascot="default" ...>` + "메뉴 보러 가기" 인라인 CTA (`navigate('/menu')`). sticky CTA는 미렌더.
  - **List**: `<ul>` + `<CartItem>` 매핑 + sticky CTA(합계 PriceTag + "주문 정보 입력" 버튼).
  - **수량 0 전이 = remove**: `<CartItem onRemove={removeItem} onChangeQty={changeQty}>` — CartItem 자체 분기(수량 1에서 − → onRemove 호출, Task 2.6 회귀)에 위임.
- 헤더 `h2` 사용 — CustomerLayout 의 사이트 헤더가 시각상 h1 위치라 페이지는 h2 + list 내 CartItem h3 일관성 (axe heading-order 통과).

### CheckoutPage.jsx (119줄)

- **useState controlled** (react-hook-form 미설치) — `studentId`/`name`/`isExternal`/`deliveryType`/`tableNo`/`useCoupon`/`touched`/`submitting`/`submitError`.
- **조건부 필드**는 `isExternal`/`deliveryType` 분기로 조건부 렌더 (§3.5 7조) — `Controller` 패턴 X.
  - `!isExternal && <학번 input>`
  - `!isExternal && <쿠폰 체크박스>` (외부인은 쿠폰 X — ADR-021)
  - `deliveryType === 'dineIn' && <테이블 input>`
- 학번 정규식 `^\d{2}\d{2}37\d{3}$` (ADR-019 변경). 비숫자 자동 strip (`replace(/\D/g, '')`).
- 검증은 **render 중 계산** — `errors`/`isValid` 모두 useEffect 없이 매 렌더 derive (§3.5 4조).
- 폼 제출 (`handleSubmit`):
  - 전 필드 touched 표시 → `isValid` 확인 → `apiFetch(API.ORDERS, ...)` 호출 (ADR-020 페이로드 형태).
  - `BusinessClosedError` 는 `throw` (CustomerLayout 의 `useGlobalErrorHandler` 가 catch — G13).
  - `ApiError` 는 `setSubmitError(err.message)` → `<ErrorState variant="inline-field">` 인라인 렌더.
- sticky CTA: `<Button loading={submitting}>주문 접수</Button>` — 제출 중 LoadingState inline (3분기 처리 Loading).

## 검증한 결정

- **§3.5 1조** 페이지 ≤120줄 — 양 페이지 모두 통과 (회귀 테스트 강제).
- **§3.5 2조** Zustand 셀렉터 — `useCartStore(cartSelectors.totalPrice)` 패턴 (CartPage·CheckoutPage 모두).
- **§3.5 4조** render 중 derive — CheckoutPage `errors`/`isValid` useEffect X.
- **§3.5 7조** 조건부 필드 — `isExternal`/`deliveryType` 분기 조건부 렌더.
- **ADR-019** 학번 정규식 `^\d{2}\d{2}37\d{3}$` — 6 케이스 회귀.
- **ADR-020 Pattern B** — apiFetch 페이로드 `items[{menu_id, quantity}]` + `coupon` 만, 가격 X.
- **ADR-021** 외부인 — `student_id: null` + `is_external: true` + 쿠폰 미렌더.
- **G11** 인벤토리 — 헤더 "🎒 인벤토리 (N개)".
- **G13** BusinessClosedError throw → CustomerLayout 의 useGlobalErrorHandler 위임 (CheckoutPage `handleSubmit` 의 `if (err instanceof BusinessClosedError) throw err;`).
- **UX §6.1** Empty 마스코트 — `<EmptyState mascot="default">`.
- **UX-1** 토스트 X — Empty/Error 모두 인라인 CTA / 인라인 ErrorState.
