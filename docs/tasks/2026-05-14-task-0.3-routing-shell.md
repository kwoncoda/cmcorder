### 결정 근거

- **`AppRoutes` named export 분리** — 라우터(BrowserRouter) 와 라우팅 트리(Routes) 를 분리해 테스트는 MemoryRouter 로 격리 가능. `App` default export 는 BrowserRouter 로 감싸기만 함. DRY 만족 + 테스트 격리성.
- **`PageLoading` named export** — Suspense fallback 컴포넌트를 export 해서 단위 테스트로 직접 검증 가능. 향후 Phase 2.11 에서 LoadingState 로 교체하기 쉬움.
- **ErrorBoundary 의 fallback 함수형 지원** — class API 의 `componentDidCatch` 가 잡은 error 를 fallback 으로 흘려보내려면 함수형이 필요. 사용 측: `<ErrorBoundary fallback={({error}) => <ErrorPage code={500} message={error?.message}/>}>`.
- **사용자 / 관리자 import 전략 분리** — 사용자는 정적 import (즉시 진입 보장, 학생회 외 모든 트래픽), 관리자는 `React.lazy()` (학생회 5명만 진입). USER_FLOW §3.5 8조 권장.
- **smoke.test.jsx 삭제** — 환영 메시지 검증이 라우팅 셸로 교체되어 무효화됨. App.test.jsx 첫 케이스 (`/` → menu-page) 가 부트스트랩 회귀를 대체.
- **Suspense fallback 테스트 우회** — jsdom + Vite test 환경에서 dynamic import 가 microtask 안에 동기 resolve 되어 fallback 캡쳐 타이밍이 불안정. 대신 (a) PageLoading 컴포넌트 자체 회귀, (b) 영구 pending lazy 를 직접 Suspense 에 넣어 fallback 노출 검증. React 18 Suspense 동작 자체를 검증하기에 충분.
- **`/` → `/menu` redirect** — `<Navigate to="/menu" replace />` 로 history 누적 없이 단방향. 첫 진입 시 메뉴 노출이 기본 UX (홈 화면 따로 없음).
- **404 catch-all** — `Route path="*"` 마지막. ErrorPage code=404. `/closed` 영업 외 redirect 가드는 placeholder 만 — 실제 가드 로직은 Phase 3/4 에서.
