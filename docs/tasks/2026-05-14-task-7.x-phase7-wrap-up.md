## 한 일

### 1. 부록 D 회귀 자동화

`src/__tests__/appendix-d.test.js`에 5 케이스를 작성. D.7은 useEffect 안의
정당한 redirect(401·404)는 화이트리스트로 허용하고 그 외 새 부수효과는
실패하도록 설계.

### 2. KeyboardHelpModal organism

`BoothMinimapModal`과 동일한 패턴(latest ref + Esc + 포커스 복귀)을 따라
간결하게 구성. 단축키 4종(Enter/Esc/Tab/?)을 키보드 키캡 스타일로 표시.

- backdrop(검은 절반투명 버튼) 클릭 시 닫힘 — `aria-label="단축키 안내 닫기"`
- 닫기 버튼 클릭 / Esc 키 입력 시 닫힘
- Button atom의 forwardRef로 초기 포커스 이동

### 3. DashboardPage 통합

`?` 키 입력 시 `setShowHelp((s) => !s)` 토글. input/textarea/select/
contenteditable 포커스 시에는 토글 금지(타이핑 보호). 헬프 모달은 CLOSED 화면
및 OPEN 화면 모두에서 표시 가능.

DashboardPage가 124줄로 늘었던 것을 헤더 주석 압축으로 116줄까지 줄여 D.8 회귀 통과.

### 4. 운영 문서

`admin-card.md`는 인쇄용 한 장 카드(가동·운영·정산·비상 4 섹션 + 외부인 흐름).
`d1-rehearsal.md`는 10 섹션 체크박스로, 5/19 화요일 리허설 시 운영진이
한 줄씩 체크하며 따라가는 형식.
