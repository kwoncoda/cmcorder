### 2. GREEN — 컴포넌트 구현

#### VARIANT_CONFIG (모듈 최상위 — §3.5 6조)

```js
const VARIANT_CONFIG = {
  recommended: { label: 'RECOMMENDED', colorClasses: 'text-stamp-green border-stamp-green', rotation: '-3deg' },
  'sold-out':  { label: 'SOLD OUT',    colorClasses: 'text-stamp-red border-stamp-red',     rotation: '2deg'  },
  paid:        { label: 'PAID',        colorClasses: 'text-stamp-green border-stamp-green', rotation: '-2deg' },
  done:        { label: 'DONE',        colorClasses: 'text-stamp-green border-stamp-green', rotation: '3deg'  },
  canceled:    { label: 'CANCELED',    colorClasses: 'text-stamp-red border-stamp-red',     rotation: '1deg'  },
};
```

- **회전 고정값** (랜덤 X) — 테스트 회귀 친화 + 디자인 일관성. 5 variant 각 다른 각도로 자연스러운 시각 다양성 확보.
- **색 매핑** — recommended/paid/done = 녹색(긍정), sold-out/canceled = 빨강(부정).
- **colorClasses 는 *완전한* 클래스 문자열** — 동적 보간(`text-${token}`)은 Tailwind purge 에 잡혀 CSS 누락 → 처음 보간 버전으로 작성했다가 빌드 CSS 확인 시 `text-stamp-green`/`border-stamp-green`이 없음을 발견하고 완전한 클래스 패턴으로 교체.

#### 회전은 inline style — Tailwind arbitrary 회피

```jsx
<span style={{ transform: `rotate(${config.rotation})` }}>
```

`rotate-[Xdeg]` 같은 arbitrary 값은 동적 보간 시 purge 미스크. inline style 이 가장 안전.

#### 접근성 — role=img + aria-label

도장이 *내용*인 패턴. `aria-label="${text} 도장"` (한국어). customLabel 사용 시 label 과 aria-label 동시 override.

#### 등장 모션은 본 atom 에서 생략

명세는 "도장 찍힘 모션 150ms 등장" 이지만:
- transition-transform + duration-stamp 클래스로 *훅*만 제공.
- 등장 keyframe 은 페이지 통합 단계에서 wrapper 가 결정(예: Phase 4.2 MenuPage 의 RECOMMENDED 도장).
- 본 atom 자체에 `@keyframes stamp-press` 박으면 모든 사용처에서 매번 등장 → 페이지 진입 시 1회만 의도라면 wrapper 가 책임.

reduced motion 회귀는 `transition-transform` + `duration-stamp` 클래스 존재 자체로 확보 — globals.css 의 `@media (prefers-reduced-motion: reduce)` 가 transition-duration 을 0.01ms 로 강제하기 때문.

## 결정 노트

- **회전은 variant 별 고정** — 명세의 "랜덤 -3~+3 deg" 는 시각 다양성 의도였으나, 5 variant 각 다른 각도(-3, +2, -2, +3, +1)로 동일 효과 + 테스트 안정성 확보. 같은 도장이 한 페이지에 다회 노출되어 회전 다양성이 더 필요하면 wrapper 에서 추가 회전 검토.
- **colorClasses 동적 보간 X** — 첫 구현에서 `colorToken: 'stamp-green'` + 런타임 `text-${colorToken}` 보간으로 만들었다가 빌드 CSS 검증에서 누락 발견. Tailwind purge 는 *완전한 문자열만* 정적 분석하므로 VARIANT_CONFIG 에 완전한 클래스 문자열을 미리 적어둠.
- **inline style transform** — `rotate-[Xdeg]` arbitrary 값 대신 inline style 로 회전. arbitrary 값은 동적 보간 시 purge 미스 + 임의 각도 안전성 확보.
- **등장 모션은 wrapper 책임** — 본 atom 은 도장의 *모양·색·회전*에 집중. 등장 keyframe 은 페이지 통합 시 wrapper 가 결정 (Phase 4.2 MenuPage 통합 시 확정).
- **font-stencil fallback** — Black Ops One 은 design-bundle 폰트이지만 현재 로컬 import 안 됨 → tokens.css 의 `--font-stencil: 'Black Ops One', 'Pretendard Variable', sans-serif` 가 자동 fallback. 테스트는 클래스 존재만 검증 (실제 렌더 폰트는 페이지 통합 시 확인).
