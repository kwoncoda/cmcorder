## 한 일

### MenuAdminPage (Task 5.4)

1. `useApi(({ signal }) => apiFetch('/admin/api/menus', { schema: MenuListSchema, signal }), [])` — 첫 진입 단발.
2. `optimistic` 상태 `{ [menuId]: { soldOut?, recommended? } }` — 베이스 메뉴 위에 spread 병합으로 화면 렌더.
3. `togglePatch(id, patch)`:
   - 낙관 적용 → `apiFetch(API.ADMIN_MENU_TOGGLE(id), { method: 'POST', body: patch })`.
   - 실패 시 해당 id의 낙관 키만 `delete` → 원상 복귀 + `setActionError(err.message)`.
4. Wrapper 함수로 Loading/Error/Empty/목록 분기 모두에 `data-testid="admin-menu-page"` 유지 — `App.test` 의 라우팅 회귀 보호.
5. 401 시 useEffect로 navigate (render 중 navigate 금지).

### SettlementPage (Task 5.5)

1. 인라인 `SettlementSchema` (zod) — `is_closed` 옵션 처리 (서버 미구현 시점 방어).
2. 정산 요약 카드 — 총 주문 수 · 총 매출 (`PriceTag` font-mono) · 진행 중 주문.
3. `canClose = in_progress_count === 0 && !is_closed` — 마감 버튼 disabled 결정. **ADR-012 가드** 그대로 UI에 반영.
4. `handleClose()`:
   - `apiFetch(API.ADMIN_SETTLEMENT_CLOSE, { method: 'POST', body: {} })` → `setStatus('CLOSED')` (G13) + `query.refetch()`.
   - 실패 시 `setCloseError(err.message)` — 인라인 alert.
5. `handleDownloadZip()` — `window.open(API.ADMIN_SETTLEMENT_ZIP, '_blank')` — 새 탭 GET. 브라우저가 Content-Disposition으로 자동 다운로드 (백엔드 6.x에서 헤더 구현 예정).
6. Wrapper 함수로 분기마다 `data-testid="admin-settlement-page"` 유지.
