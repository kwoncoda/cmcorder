## 한 일

### 1. pricing.js — ADR-020 ★★★ (서버 자체 계산)
- 시그니처가 `{items, coupon}`만 받음 → 클라이언트가 가격을 보낼 채널 자체를 제거
- 각 `menu_id`를 DB lookup으로 가격 확정, `quantity`와 곱해 합산
- 쿠폰 사용 시 10% 할인 (`Math.floor`로 정수 원 보존, ADR-020 정수 원 가드)
- `items_priced[i]` 스냅샷에 `name`·`base_price` 포함 (G10 본명 보존)
- `PricingError`(`code: MENU_NOT_FOUND` / `MENU_SOLD_OUT` / `PRICING_ERROR`) 분류
- 회귀 4 케이스 + 추가 9 케이스 = **총 13 통과**

### 2. coupon.js — ADR-019 변경
- 정규식 `^\d{2}\d{2}37\d{3}$` — prefix `202637` 강제 해제, 학과 코드 `37`만 강제
- 4단계 검증: format(정규식)·department(정규식 포함)·name(trim)·duplicate(`used_coupons` SELECT)
- `consumeCoupon` UNIQUE 위반 시 `SQLITE_CONSTRAINT_UNIQUE` → `CouponError(ALREADY_USED)`로 변환 (race 가드)
- **총 12 통과** (학번 4 패턴 + 형식·학과·이름·중복·trim·소비)

> CLAUDE.md의 "학번 prefix 202637" 기술은 ADR-019 원본 문구. 본 Task의 사용자 brief가 "ADR-019 변경"으로 학과 코드 37로 축소 결정. CLAUDE.md 우선순위에 따라 사용자 지시(=brief)가 ADR 원본보다 상위.

### 3. order-state.js — ADR-025 (13 합법 + 5 불법)
- `LEGAL_TRANSITIONS` 딕셔너리로 합법 전이만 enumerate. 터미널 상태(DONE·CANCELED) 빈 배열
- `canTransition`(boolean) + `transition`(throw) 두 함수로 호출부 선택
- 13 합법 전이 `it.each` 통과 + 합법 총 13개 회귀 카운트 + 5 불법 거부 + 메타데이터(from·to·code) = **총 20 통과**

### 4. business-state.js — G13 신규
- 2-state 머신 (OPEN ↔ CLOSED). `db.transaction()`으로 같은 행 read-modify-write 원자성
- `openBusiness`·`closeBusiness` 모두 멱등 (같은 상태 재호출 시 변경 없이 통과)
- `openBusiness` 멱등 시 `operating_date`도 보존 (테스트 6 케이스: "이미 OPEN이면 5/20 유지")
- `changed_at`는 `datetime('now')` SQL 함수로 갱신 — 상태 변경 시에만
- 단일 행 강제 (`init.sql` CHECK id=1) — id=2 INSERT 시도 시 SQLITE_CHECK throw 확인
- **총 11 통과**

### 5. settlement.js — ADR-012 + G13 자동 트랜잭션
- `canCloseSettlement` — in_progress 6상태(ORDERED·TRANSFER_REPORTED·PAID·COOKING·READY·HOLD) COUNT 0인지
- `getSettlementSummary` — DONE 주문 합계 + in_progress count + `is_closed` 플래그
- `closeSettlement` — 단일 트랜잭션에서 가드 → `settlements` INSERT → `closeBusiness(db)` 호출
  - G13 핵심: 가드 실패 시 settlements도 business_state도 ROLLBACK (테스트 검증)
- 중복 마감 거부(`ALREADY_CLOSED`) — `settlements.operating_date UNIQUE` 사전 검사
- **총 8 통과** (canClose 3 + summary 2 + closeSettlement 3)

### 6. popularity.js — 결정 E (정적 BEST)
- `recommended=1 AND sold_out=0` 우선, 부족 시 같은 조건으로 첫 N개 fallback
- 품절 메뉴는 영원히 제외
- `limit=0` 빈 배열 반환 (가드)
- **총 5 통과**

### 7. transfer-matching.js — 4요소 일치
- 순수 함수 (DB 의존 X) — 호출부에서 후보 배열을 미리 SELECT
- 이름: `use_other_name=1` 분기 시 `other_name` 사용
- 은행: `bank` 또는 `custom_bank` (기타 은행 입력 케이스)
- 금액: 정확 일치
- 시각: `Math.abs(diff) <= 300_000ms` (±5분)
- **총 8 통과**
