## 목표

본부 대시보드 헤더에 사용할 영업 상태 시각 표시 2종 (organism):

- **BusinessStateBadge** — 상태 배지. 3 시각 변형:
  1. `status='OPEN'` → 🟢 영업 중 (녹·정적)
  2. `status='CLOSED' + shouldBeOpen=false` → 🔴 영업 외 (빨·정적)
  3. `status='CLOSED' + shouldBeOpen=true` → 🔴 영업 외 (빨·**깜박**)
- **StartBusinessCTA** — CLOSED 시 보이는 큰 형광 옐로 "장사 시작" CTA.
  `status='OPEN'` 이면 렌더 X. shouldBeOpen 에 따라 primary/secondary 톤.

**결정 h (절대 깨지면 안 됨):** 깜박은 *세션 X, 매번 재생*.
sessionStorage 사용 X — 사장님이 화면을 새로 열 때마다 즉시 인지하도록.
reduced motion 시 정적 (components.css `@media`).

### 2. GREEN — 구현

#### components.css 확장

- `@keyframes business-badge-blink` — 1.2s ease-in-out infinite, opacity 50% 까지.
  `.business-badge-blink` 클래스로 BusinessStateBadge 가 조건부 부착.
- `@keyframes start-business-cta-press` — 200ms ease-out, scale(1) → scale(0.96) → scale(1).
  `.start-business-cta-press` 클래스. JS가 setTimeout 200ms 후 클래스 해제 → 단발 모션.
- `@media (prefers-reduced-motion: reduce)` 블록 확장 — 두 keyframes 모두 `animation: none`.

#### BusinessStateBadge.jsx 핵심

- **3 시각 상태 결정 트리**:
  - `isOpen = status === 'OPEN'` (방어 — 'OPEN' 외 모두 CLOSED 취급, fallback 보장)
  - `needsAttention = !isOpen && shouldBeOpen`
  - `colorCls`: OPEN → `bg-success text-card-ink` / 나머지 → `bg-danger text-card-ink`
    needsAttention 인 경우 `business-badge-blink` 추가
- **시맨틱**: `role="status"` + `aria-live="polite"` — 상태 변경 시 스크린리더가 부드럽게 알림.
- **forwardRef** — 호출자가 span DOM 직접 접근 가능.
- **이모지 + 텍스트**: 🟢/🔴 + "영업 중 (OPEN)" / "영업 외 (CLOSED)".

#### StartBusinessCTA.jsx 핵심

- **OPEN 단락 평가**: `if (status === 'OPEN') return null;` — 이미 영업 중이면 렌더 X.
- **Button atom 재사용 (DRY)** — variant=primary/secondary + size=lg + block + loading.
  pressed=true 시 className에 `start-business-cta-press` 추가.
- **클릭 흐름**: `setPressed(true)` → `setTimeout(200ms, setPressed(false))` → `onStart?.()`.
  pressed 클래스가 즉시 추가되어 모션 트리거, 200ms 후 해제 (단발 — sessionStorage X).
- **shouldBeOpen=false** → secondary 톤 + 작은 안내 카피 "아직 영업 시작 시간이 아닙니다".
- **에러 인라인**: `<p role="alert" data-testid="cta-error">` — 호출자가 API 실패 메시지 전달.
- **aria-label 동적**: shouldBeOpen 에 따라 "지금 영업 시작" / "영업 시간 전" — 스크린리더 명확화.

### 3. 테스트 보정

처음 1차 실행에서 1 실패:

> `secondary variant 검증` — `expect(btn.className).not.toMatch(/bg-accent/)` 가 실패.
> Button atom 의 `secondary` variant 가 `border-accent`, `hover:bg-accent/10`,
> `focus-visible:outline-accent` 등 "bg-accent" 부분 문자열을 포함.

**보정:** 단어 경계 패턴 `/(^|\s)bg-accent(\s|$)/` 으로 *채워진 단독 클래스*만 매칭하도록 변경.
secondary 잔재(`border-accent` 등)와 충돌 회피.

추가로 secondary 확인 강화 — `bg-transparent` 시그니처 클래스 존재도 검증.

### 회귀 안전망 핵심

- **깜박 클래스 회귀**: `CLOSED + shouldBeOpen=true` → `business-badge-blink` 필수 매칭 (결정 h).
  세션 영속화로 회귀 시(예: sessionStorage 도입), 이 테스트는 그대로 통과하나 — 명세 위반은
  코드 리뷰에서 잡혀야 함. 단위 테스트는 단발 재생이라는 의미보다 *클래스 부착*을 검증.
- **OPEN 시 CTA 미렌더**: `container.firstChild === null` — 명세 위반(렌더됨) 시 즉시 fail.
- **200ms scale 모션**: `vi.useFakeTimers` + `act(vi.advanceTimersByTime(200))` 로
  setTimeout 결정성 보장. pressed 클래스 추가 → 200ms 후 제거 양방향 검증.
