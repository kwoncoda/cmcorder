## 한 일

### 1. 의존성 추가

`package.json` dependencies:

- `express@^4.22.2` — **명세 ADR-024 Express 4 준수**. `npm i express` 기본 설치가 v5.2를 가져와서 5에서 4로 수동 다운그레이드(미들웨어 서명·라우터 동작 차이 회귀 회피).
- `helmet@^8.1.0`, `pino@^10.3.1`, `pino-http@^11.0.0`, `dotenv@^17.4.2`.

devDependencies:

- `supertest@^7.2.2` — listen 없이 Express 인스턴스 직접 호출.

### 2. 파일 작성 (TDD: RED → GREEN)

1. **RED**: `server/__tests__/healthz.test.js` 먼저 작성 → `npm test` 시 `Failed to resolve import "../app.js"`로 1 스위트 실패 (단위 31 통과).
2. **GREEN**: `server/lib/logger.js` → `server/app.js` → `server/server.js` 순서로 작성. 재실행 시 5 스위트/33 통과.

### 3. 설정 변경

- `vite.config.js` `test.include`에 `server/**/*.{test,spec}.{js,jsx}` 추가 → Vitest가 server 테스트도 수집.
- `package.json` scripts: `server`, `server:watch` (node --watch) 추가.

### 4. 결정 근거

- **app.js vs server.js 분리**: 테스트는 listen 없이 인스턴스만 만들어 supertest로 호출(포트 충돌·부수효과 없음). server.js는 listen + graceful shutdown만 담당.
- **CSP off**: helmet 기본 CSP는 SPA 정적 호스팅(Phase 6)에서 인라인 스크립트/번들 경로와 함께 결정해야 정확하므로 부트스트랩 단계에서 비활성화.
- **/healthz autoLogging ignore**: 부스 모니터링이 빈도 높게 호출할 헬스체크가 pino 로그를 채우지 않도록 제외.
- **pino base.pid/hostname 제거**: 부스 운영 시 한 줄 로그 가독성 우선.
- **JSON limit 64KB**: 주문/메뉴 payload는 모두 수백 바이트 수준. 과도한 body 차단.
