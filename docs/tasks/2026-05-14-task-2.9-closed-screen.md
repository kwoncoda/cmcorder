## 한 일

### API
```jsx
<ClosedScreen
  reason="before-open"        // 'before-open' | 'after-close' | 'after-settlement' | 'both-days-done'
  operatingDate="2026-05-20"  // YYYY-MM-DD
  onRefresh={() => window.location.reload()}

```

### REASON_CONFIG (모듈 SoT)
- `before-open` → "아직 영업 시작 전이에요" / default 마스코트(🪖)
- `after-close` → "오늘 영업이 끝났어요" / arrive 마스코트(🎉)
- `after-settlement` → "오늘 부스 정산 마감!" / arrive 마스코트(🎉)
- `both-days-done` → "축제 부스가 종료되었습니다" / canceled 마스코트(😢)
- 알 수 없는 reason → `'before-open'` fallback (`?? REASON_CONFIG['before-open']`)

### OPERATING_SCHEDULE (모듈 SoT — UX §8.8)
- 2026-05-20 (수) 16:30 ~ 21:00
- 2026-05-21 (목) 11:00 ~ 21:00

### 접근성
- `<section role="region" aria-labelledby="closed-title">` — 풀스크린 안내 의미.
- 운영 일정 카드: `role="status"` + `aria-live="polite"` (UX §8.8 자동 announce 회귀 보호).
- 새로고침 버튼: `aria-label="화면 새로고침"`.

### 운영 일정 강조 색 결정 근거 — AI 슬롭 #26 충돌 검토

명세 §2.9 본문이 `text-accent font-semibold` 사용을 제안하면서 동시에 AI 슬롭 #26 ("본문 콘텐츠 형광 옐로 텍스트 금지")과 약한 충돌이 있다고 언급. 작업자 판단으로 본 구현은 **명세 코드 그대로 `text-accent font-semibold` 유지** — 근거:

1. **컨텍스트:** 운영 일정 카드는 *데이터 카드*(`role="status"`)이지 본문 콘텐츠가 아님. AI 슬롭 #26은 단락·헤딩 등 본문 텍스트의 형광 색 사용을 막는 가이드.
2. **의미 가산:** 강조 의도가 "오늘이 어느 날인지" 명확. 시각 구분 필수.
3. **a11y:** `font-semibold` 만으로는 색약·저시력 사용자의 변별이 약함 → accent 색이 보강 역할.
4. **회귀:** 테스트는 `li.text-accent` 또는 `li.font-semibold` 둘 다 받아주도록 작성 — 향후 디자인 결정 변경 시 코드만 바꾸면 됨.
