# Task 6.5 / 6.6 / 6.8 — Repositories + 사용자 API + Middleware

## 목표
백엔드 통합 첫 단계 — DB 레이어(Repositories), 사용자 핵심 API 6개, 영업 가드·인증·에러 미들웨어를 한 번에 결합한다. ADR-020 Pattern B(가격 자체 계산)·ADR-019(쿠폰 UNIQUE)·G13(영업 외 가드)을 회귀 보호.

## 만든 것

### Repositories (server/repositories/)
- `menu-repo.js` — `listMenus` · `getMenu` · `toggleMenu(soldOut/recommended/base_price)`
- `order-repo.js` — `nextOrderNo`(일자 시퀀스) · `createOrder`(orders+order_items 트랜잭션) · `getOrder` · `listOrders` · `updateOrderStatus`(timestamp 자동) · `updateTransferInfo`
- `coupon-repo.js` — `hasCouponBeenUsed` · `listUsedCoupons` · `countUsedCoupons` (UNIQUE 위반 변환은 domain/coupon.js)
- `business-state-repo.js` — domain/business-state.js wrapper (`getCurrentState` · `openBusinessDay` · `closeBusinessDay`)

### Middleware (server/middleware/)
- `business-state.js` — `businessStateGuard(db)` factory. `/admin/*` 통과, GET 통과, 사용자 POST/PUT/DELETE 시 CLOSED 상태면 HTTP 423 `BUSINESS_CLOSED`
- `admin-auth.js` — `requireAdmin` (req.session.adminId 가드, 401)
- `error.js` — `errorHandler` 전역. ZodError(400) · PricingError(400) · CouponError(400) · StateTransitionError(409) · SettlementError(409) · 기타(500)

### Routes (server/routes/)
- `customer.js` — 12 엔드포인트 중 핵심 6개:
  - `GET /api/menus` (snake_case → camelCase 시리얼라이즈)
  - `GET /api/popular`
  - `GET /api/business-state`
  - `POST /api/orders` — **ADR-020 Pattern B**: 클라가 보낸 total/items_priced 무시, server-side calculatePrice + 쿠폰 동시 consumeCoupon
  - `GET /api/orders/:id`
  - `POST /api/orders/:id/transfer-report`
- 모든 POST body는 zod 스키마 검증

### app.js 갱신
- db 주입 시 `businessStateGuard` + `customerRoutes` 마운트
- 에러 핸들러를 domain-aware `errorHandler`로 교체 (기존 단순 500 → 분기 매핑)

### 결정 근거
- **Coupon-repo**는 검증·UNIQUE 위반 변환을 도메인이 이미 담당하므로 단순 조회 helper만 노출. 호출자는 `consumeCoupon` (domain)을 직접 사용 — Pattern B 원칙대로 검증 흩어짐 방지.
- **Business-state-repo**는 도메인 모듈을 라우트 친화적 이름으로 래핑만. 추가 검증 없이 thin wrapper.
- **createApp({ db })** — db 없으면 영업 가드·라우터 마운트 X (healthz 단독 테스트와의 하위 호환).
- **zod**는 이미 deps에 있음. 추가 설치 X. `express-session`은 Task 6.7에서 필요 — 본 Task에서는 admin-auth가 req.session만 의존하도록 분리(설치 X).
- **이체 신고 zod**: `customBank`·`useOtherName`·`otherName`은 optional. 422 → 400 합쳐 단순화.
