### Vite envDir 격리 — 핵심 디버깅

**증상:** production 빌드 시 axe-core 가 메인 chunk 에 들어가고, `jsxDEV`/`fileName`
주석까지 dev 메타데이터가 남음 (165 KB → 589 KB).

**원인:** Vite 5 의 `resolveConfig` 가 `.env` 의 `NODE_ENV=development` 값을 보고
`process.env.NODE_ENV` 를 development 로 강제 설정 — *plugin-react 가 dev 빌드*.
프로젝트 `.env` 는 백엔드 Express(Task 0.5) 용이라 변수를 제거할 수 없음.

**해결:** `vite.config.js` 에 `envDir: 'config/vite-env'` 지정. 빈 디렉터리라
Vite 는 어떤 `.env` 도 못 읽고 production 빌드를 정상 수행. 결과: 165.61 kB.

```js
// vite.config.js
envDir: resolve(__dirname, 'config/vite-env'),
```

또한 bundle.test.js 에서 `execSync('npm run build')` 시 *vitest 가 부모 process 에
NODE_ENV=test 를 주입*하므로 자식 env 를 `NODE_ENV=production` 으로 명시 override.
