### 2. GREEN — 5 컴포넌트 구현

#### 2-1. PriceTag

```jsx
<PriceTag value={6000} />              → "6,000원"
<PriceTag value={6000} unit="none" />  → "6,000"
<PriceTag value={6000} strikethrough />  → line-through 클래스
<PriceTag value={6000} negative />       → "-6,000원" + text-danger
```

- 핵심: `new Intl.NumberFormat('ko-KR')` 모듈 최상위 1회 생성 (재호출당 인스턴스 생성 회피 — 성능).
- `font-mono tabular-nums` 결합 — JetBrains Mono `tnum` 활성화. 가격 줄맞춤 보장.
- `SIZE_CLASSES` 모듈 최상위 (sm: text-xs, md: text-sm, lg: text-lg). 동적 보간 X.

#### 2-2. StatusChip

```jsx
<StatusChip status="ORDERED" />     → ⏳ 주문 접수 (info)
<StatusChip status="COOKING" />     → 🔥 조리 중   (accent)
<StatusChip status="DONE" />        → 🎉 수령 완료 (success)
```

- 8 상태 모두 모듈 최상위 `STATUS_CONFIG` 매핑(label·icon·cls).
- 시맨틱 색 정렬 — ORDERED=info, TRANSFER/HOLD=warning, PAID/DONE=success, COOKING/READY=accent, CANCELED=danger.
- 이모지는 `<span aria-hidden="true">` 래핑 — 텍스트만 SR 읽음.
- 알 수 없는 status fallback: ORDERED.
- size 'sm'(text-2xs)·'md'(text-xs) — 완전 문자열.

#### 2-3. CountBadge

```jsx
<CountBadge count={3} />               → "3"
<CountBadge count={99} max={9} />      → "9+"
<CountBadge count={0} hideZero />       → null
<CountBadge count={3} label="장바구니 항목" />  → aria-label="장바구니 항목 3개"
```

- 핵심: `count === 0 && hideZero` 시 `return null` — 빈 배지 자국 제거.
- `count > max` 시 `${max}+` 형식.
- `bg-accent text-card-ink` — 형광 옐로 배지 (장바구니 신호).
- `font-mono tabular-nums` — 숫자 폭 균일.

#### 2-4. IconLabel

```jsx
<IconLabel icon={<Map size={16} />} text="부스 약도" />  // ReactNode
<IconLabel icon="🗺️" text="부스 약도" />              // 자동 aria-hidden
<IconLabel icon="🗺️" text="약도" gap="sm" inline />
```

- 핵심: `typeof icon === 'string'` 분기 — 문자열은 자동 `<span aria-hidden="true">` 래핑, ReactNode는 그대로 렌더.
- `gap` 'xs'(2px)·'sm'(8px)·'md'(12px) — spacing 토큰 매핑.
- `inline` true 시 `inline-flex`, false 시 `flex` (블록).

#### 2-5. MenuFallback

```jsx
<MenuFallback category="chicken" name="자라쥬" />  → 🍗 + "자라쥬"
<MenuFallback category="side"    name="감자튀김" />  → 🍟
<MenuFallback category="drink"   name="콜라" />      → 🥤
<MenuFallback category="unknown" name="???" />     → ❓
```

- `CATEGORY_EMOJI` 매핑 모듈 최상위 — ADR-006 분류와 1:1 정합.
- `role="img"` + `aria-label="{category} {name} 일러스트 (대체)"` — 일러스트가 *내용*인 패턴.
- 이모지 자체는 `<span aria-hidden="true">` — 부모 aria-label이 의미 전달.

## 결정·트레이드오프

1. **`Intl.NumberFormat` 모듈 최상위 캐싱** — 렌더당 인스턴스 생성 회피. ICU 기반이라 cold init이 100us 수준이라 작지만, 메뉴 카드 다수 동시 렌더 시 누적 — 명시적으로 1회 생성.

2. **CountBadge `hideZero` null 렌더 패턴** — `display: none` 대신 React 트리에서 완전히 제거. 부모 layout 의 `gap`이 빈 배지 자국으로 어긋나는 회귀 회피.

3. **MenuFallback 의 `role="img"` 결정** — 이모지가 *내용* (a11y 의미 단위). 일반 IconLabel처럼 decorative 처리하면 SR 사용자가 메뉴 카테고리 인지 불가. ADR-006 정신(분류 인지)을 a11y 까지 확장.

4. **StatusChip 시맨틱 색 분배** — 8 상태를 5 시맨틱 토큰(info/warning/success/accent/danger)에 분배. ORDERED·DONE이 같은 success가 되면 혼동 → ORDERED는 info(시작 단계 신호), DONE은 success(긍정 완료).

5. **이모지 자동 aria-hidden 패턴** — IconLabel은 `typeof icon === 'string'` 으로 분기. lucide-react SVG는 ReactNode → 호출자가 Icon atom으로 a11y 책임. 이중 책임 X.
