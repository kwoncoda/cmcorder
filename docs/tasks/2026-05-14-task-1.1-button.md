# Task 1.1 — Button atom (5 variant · 3 size · 모바일 hitbox · a11y)

## 목표

- IMPLEMENTATION_PLAN.md §1.1 / COMPONENT_GUIDE.md §2.1 완수.
- primary(형광 옐로) · secondary(외곽선) · ghost · danger 4 variant + disabled prop.
- 3 size(sm 44 / md 48 / lg 56) — 모바일 터치 hitbox 보장 (44px 미만 금지).
- block(full-width) · loading(spinner + aria-busy=true + disabled 자동) · forwardRef.
- DESIGN.md §12.3 focus-visible 토큰 — 형광 옐로 outline + offset 2px.
- axe-core a11y 위반 0건.

## 만든 것

- `src/components/atoms/Button.jsx` — forwardRef 컴포넌트 + Spinner 헬퍼.
- `src/components/atoms/__tests__/Button.test.jsx` — 17 케이스(variant 4 · size 3 · click · disabled · loading · block · ref · 기본값 · a11y · passthrough · type=submit · focus-visible).

## 한 일

### 1. RED — 테스트 먼저

`Button.test.jsx` 17 케이스 작성. 핵심 검증:

- `it.each(['primary','secondary','ghost','danger'])`로 4 variant 렌더링.
- `it.each([['sm','44'],['md','48'],['lg','56']])`로 3 size 의 `min-h-[<height>px]` 클래스 매칭.
- loading 시 — `toBeDisabled()` + `aria-busy=true` + 자식 텍스트 사라짐(`queryByText('버튼')` null) + onClick 호출 X.
- forwardRef — `createRef()` → `ref.current instanceof HTMLButtonElement`.
- focus-visible — `focus-visible:outline-accent` + `focus-visible:outline-offset-2` 클래스 명시 매칭 (DESIGN §12.3).
- axe-core — `await axe(container)` → `toHaveNoViolations()`.
- passthrough — `aria-label`, `data-testid` 전달.

RED 확인: `npm test src/components/atoms/__tests__/Button.test.jsx` → `Failed to resolve import "../Button.jsx"`.

### 2. GREEN — Button 구현

`src/components/atoms/Button.jsx`:

- `forwardRef(function Button({ variant='primary', size='md', block, loading, disabled, type='button', className, children, ...rest }, ref) { ... })`.
- `variantClasses` 객체 — 4 variant 각각의 Tailwind class (tokens는 tailwind.config.js extend.colors에서 `bg-accent`/`text-card-ink` 등으로 매핑됨).
- `sizeClasses` — `sm: min-h-[44px]`, `md: min-h-[48px]`, `lg: min-h-[56px]`. 명세는 sm 40px 였으나 본 가이드 추가 결정으로 sm=44px(모바일 hitbox 안전).
- `baseClasses` — `inline-flex` 정렬 · `font-semibold` · `rounded-md` · `transition-all duration-tap` · `focus-visible:outline focus-visible:outline-2 focus-visible:outline-accent focus-visible:outline-offset-2` · `disabled:cursor-not-allowed disabled:opacity-50`.
- `isDisabled = disabled || loading` → 둘 다 동일하게 native `disabled` 속성 적용.
- `aria-busy={loading || undefined}` — false 시 속성 자체 제거(jest-dom 직관).
- `Spinner` — `w-4 h-4 border-2 border-current border-r-transparent rounded-full animate-spin motion-reduce:animate-none`, `role="status"` + `aria-label="로딩 중"`.

### 4. Tailwind purge 검증 (sanity)

`npm run build` 후 `dist/assets/index-*.css`에서 arbitrary value · focus-visible · animation 클래스 모두 포함 여부 확인:

```
min-h-\[44px\]            ← OK
min-h-\[48px\]            ← OK
min-h-\[56px\]            ← OK
focus-visible\:outline-accent:focus-visible   ← OK
motion-reduce\:animate-none                   ← OK
animate-spin              ← OK
bg-accent / bg-accent-pressed / bg-danger     ← OK
```

Tailwind 3 의 JIT 가 `src/**/*.{js,jsx}` 콘텐츠 스캔에서 정상 picking → safelist 불필요.

### 5. 알려진 jsdom stderr (실패 X)

axe-core 가 jsdom 환경에서 `HTMLCanvasElement.prototype.getContext` 호출 시 stderr 경고를 출력함. 이는 jsdom 의 미구현 API 경고일 뿐 테스트 자체는 통과 — axe-core 가 폰트 ligature 검사 시 canvas 측정에 fallback 하여 결과를 만들고, violations 0건을 반환. ErrorPage a11y 테스트(Task 0.3)에서도 동일한 stderr 가 이미 관측됨.
