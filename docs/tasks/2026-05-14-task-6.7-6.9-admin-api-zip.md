## 한 일

### 1. 관리자 API 16 엔드포인트 (`server/routes/admin.js`)

| Method | Path | 기능 | 가드 |
| --- | --- | --- | --- |
| POST | `/admin/login` | PIN 6자리 검증 → 세션 쿠키 | 없음 |
| POST | `/admin/logout` | 세션 destroy | 없음 |
| GET | `/admin/api/business/state` | 영업 상태 | requireAdmin |
| POST | `/admin/api/business/open` | OPEN 전환 (멱등) | requireAdmin |
| GET | `/admin/api/menus` | 메뉴 8 + soldOut/recommended | requireAdmin |
| POST | `/admin/api/menus/:id/toggle` | 부분 패치 | requireAdmin |
| GET | `/admin/api/orders` | 주문 목록 (status·date 필터) | requireAdmin |
| GET | `/admin/api/orders/:id` | 단일 주문 + items | requireAdmin |
| POST | `/admin/api/orders/:id/transition` | ADR-025 13 합법 전이만 | requireAdmin |
| GET | `/admin/api/transfers` | TRANSFER_REPORTED만 | requireAdmin |
| GET | `/admin/api/settlement` | 요약 | requireAdmin |
| POST | `/admin/api/settlement/close` | ADR-012 가드 + G13 자동 CLOSED | requireAdmin |
| GET | `/admin/api/settlement/zip` | ZIP buffer 다운로드 | requireAdmin |

핵심 결정:
- `router.use('/admin/api', requireAdmin)` 한 줄로 모든 보호 라우트 일괄 가드 → 누락 방지
- 전이는 `transition(from, to)` 호출 → 불법 시 `StateTransitionError` throw → `errorHandler`가 409로 매핑 (기존 인프라 재사용, ADR-025 회귀)
- `closeSettlement`는 도메인이 이미 ADR-012 가드 + business_state CLOSED 트랜잭션 보장 (라우트는 wrapper)
- 응답 직렬화는 `serializeMenu` helper로 SQLite 0/1 → boolean 변환 (customer 라우트와 정합)
- `status` 쿼리 — 단일 vs 다중 자동 분기 (`?status=ORDERED` vs `?status=ORDERED,PAID`)

### 2. express-session 미들웨어 (`server/middleware/admin-auth.js`)

- 메모리 store (단발 운영 — 영속성 불필요, 단일 프로세스 가정)
- `httpOnly + sameSite=lax` (CSRF 완화)
- `secure: NODE_ENV === 'production'` (HTTPS 종단 가정)
- `SESSION_SECRET` env 미설정 + production 환경이면 logger.warn 1회
- `maxAge: 12h` (운영 1일 + 여유)
- `loginAdmin(db, pin)` — `verifyPin` (timingSafeEqual) 재사용, admins 미시드면 throw

### 3. 자동 ZIP (`server/jobs/auto-snapshot.js`)

- ADR-022 변경 반영: 기본 `intervalMs = 120 × 60 × 1000` (2시간), `maxBackups = 6` (보존 12시간)
- `AUTO_SNAPSHOT_INTERVAL_MIN` / `AUTO_SNAPSHOT_ROTATE` env로 오버라이드 가능 (`system_settings` 시드와 정합)
- 회전: `mtime` 기준 최신 N개 유지, 나머지 `unlinkSync`
- WAL checkpoint(TRUNCATE)로 main DB flush → 트랜잭션 시점 일관성
- `:memory:` DB는 WAL 미지원 → try/catch + `serializeDb` (`.sql dump`) 폴백
- `archiver` v8 ESM: `import { ZipArchive }` (default export 없음)
- `createSettlementZip` — Buffer 즉시 반환 (HTTP 응답 다운로드용, `summary.json` 포함)
- `startAutoSnapshot` 반환: cleanup 함수 (`() => clearInterval`)
- 즉시 1회 실행 X — startup 부담 회피, 첫 tick은 intervalMs 후
