### 2. GREEN — 컴포넌트 구현

#### EmptyState.jsx

```jsx
<section role="status" aria-label={title} data-testid="empty-state">
  <MascotState state={mascot} size="md" useFallback />
  {title && <h2>...</h2>}
  {description && <p>...</p>}
  {actionLabel && onAction && <Button variant="primary">{actionLabel}</Button>}
</section>
```

- variant card/page 분기 (rounded-md vs min-h-screen).
- MascotState `useFallback={true}` 기본 — 결정 c (자산 미수령 단계).

#### LoadingState.jsx

```jsx
const [visible, setVisible] = useState(minimumDelay === 0);

useEffect(() => {
  if (minimumDelay === 0) return undefined;
  const t = setTimeout(() => setVisible(true), minimumDelay);
  return () => clearTimeout(t);
}, [minimumDelay]);

if (!visible) return null;
```

- `minimumDelay=0` 일 때 ESLint `consistent-return` 회피 위해 명시적 `return undefined`.
- variant card(=rounded-md) / page(=min-h-screen) / inline(=inline-flex) 3 분기.
- Spinner atom 재사용 (size sm/lg).

#### ErrorState.jsx

- `variant="inline-field"` 가 가장 다름 — `<p role="alert" className="text-danger text-xs">{title}</p>` 만 렌더 (마스코트·버튼·description 모두 미렌더).
- 그 외 (card/page) — canceled 마스코트 + code + title + description + CTA.
- `role="alert"` + `aria-live="assertive"` — 스크린리더 즉시 안내.

### 4. App.jsx PageLoading → LoadingState 위임

- 임시 `<div data-testid="page-loading" ...>로딩 중…</div>` 인라인 컴포넌트 제거.
- `LoadingState` import 추가 + `PageLoading` 을 위임 래퍼로 교체:

```jsx
export function PageLoading() {
  return (
    <LoadingState
      variant="page"
      label="로딩 중…"
      minimumDelay={0}
      data-testid="page-loading"
    />
  );
}
```

- `data-testid="page-loading"` 은 `LoadingState` 의 `...rest` 로 전달 → 기존 `App.test.jsx` 의 `getByTestId('page-loading')` 검증 호환.
- `role="status"` 는 LoadingState 가 그대로 보유 → `toHaveAttribute('role','status')` 호환.
- `"로딩 중…"` 텍스트는 `<span>` 자손에 있어 `toHaveTextContent` 통과.
