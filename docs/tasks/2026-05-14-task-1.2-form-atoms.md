### 2. GREEN — 5 atom 구현

#### Label
- `forwardRef`로 `<label>` 래핑.
- `required` prop → `<span className="text-danger ml-1" aria-label="필수">*</span>`.
- 텍스트 토큰: `text-ink font-medium text-sm mb-2xs`.

#### Input
- 베이스 클래스 상수로 분리:
  - `w-full min-h-[48px]` (모바일 hitbox)
  - `bg-card-bg text-card-ink border-2` (카드 톤)
  - `focus-visible:outline focus-visible:outline-2 focus-visible:outline-accent focus-visible:outline-offset-2` (DESIGN §12.3)
  - `placeholder:text-card-muted` (placeholder는 도움말 톤)
- `invalid` → `border-danger`, 기본 → `border-card-divider`.
- `errorId = errorMessage && id ? \`\${id}-error\` : undefined` — errorMessage 있을 때만 alert 영역 노출 + `aria-describedby`.
- `aria-invalid={invalid || undefined}` (false면 속성 제거 — Button 패턴 일관).
- `...rest` 패스 → inputMode·pattern·maxLength·value·onChange·name 자연 흐름.

#### Select
- Input과 동일 베이스 클래스 + 동일 invalid 패턴.
- `<select>{children}</select>` — native dropdown.

#### Checkbox / Radio
- 라벨 통합형: `<label htmlFor={id}>` 래퍼 안에 `<input>` + `<span>{label}</span>`.
- 래퍼 `inline-flex items-center gap-sm min-h-[56px] cursor-pointer` — 56px hitbox(UX-7).
- input: `w-5 h-5 accent-accent focus-visible:outline ...` — 형광 옐로 체크.
- Radio는 `type="radio"`만 다름.

### 6. AI 슬롭 #15 회피 확인

- Input/Select/Checkbox/Radio 어디에도 placeholder를 라벨로 사용하는 구조 X.
- `<Label htmlFor>` 가 별도 컴포넌트 — Input/Select와 1:1.
- Checkbox/Radio는 컴포넌트 안에 `label` prop으로 라벨 통합 (편의성 ↑) — 외부 `<Label>` 사용도 가능(htmlFor 자유).
- Input.test에 명시 케이스: "placeholder 는 그대로 전달되지만 라벨은 별도".
