### 결정 근거

- **컴포넌트 CSS는 분리하지 않음** — Task 명세에 따라 `components.css` / `layouts.css` 는 Task 1.x 컴포넌트 도입 시 분리. 이번 Task 는 `tokens.css` + `globals.css` 두 파일만.
- **phone / desktop 데모 클래스 미이식** — design-bundle 121~256 줄은 프로토타입 디바이스 프레임 데모용. 실제 앱에 불필요.
- **tokens.css 변경 없이 그대로** — `--color-card-muted` 는 이미 `#48402C` (2026-05-14 명도 조정값). 확인만 하고 그대로 이식.
- **CSS 변수 → Tailwind extend** 패턴 — 토큰을 single source of truth 로 유지하면서 `text-accent`, `bg-card-bg` 등 Tailwind 유틸리티도 사용 가능.
- **테스트 전략** — jsdom 은 `getComputedStyle()` 로 CSS 변수 해석이 불안정. 회귀 잠금은 `tokens.css` 원문을 `readFileSync` 로 읽어 정규식 매칭으로 대체 (단순·결정적·빠름).
