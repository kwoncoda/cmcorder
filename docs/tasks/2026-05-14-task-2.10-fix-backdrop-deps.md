## 한 일

### I-1 해결: onClose latest ref 패턴

**문제 분석:**
- `useEffect(..., [open, onClose])` — 부모가 inline 함수(`onClose={() => ...}`)
  로 매 렌더 새 함수를 전달하면 의존성 비교 실패 → cleanup → 재실행.
- 재실행 시 `previousFocusRef.current = document.activeElement` 가 *현재* 모달
  내부 요소(닫기 버튼 등)로 덮어쓰임 → 모달 닫을 때 복귀 대상 손실.
- 매 렌더 닫기 버튼으로 포커스가 끌려가는 부작용도 발생.

**수정 (BoothMinimapModal.jsx:60-66, 91):**
```jsx
// latest ref 패턴
const onCloseRef = useRef(onClose);
useEffect(() => {
  onCloseRef.current = onClose;
}, [onClose]);

useEffect(() => {
  if (!open) return undefined;
  // ...
  const onKeyDown = (e) => {
    if (e.key === 'Escape') {
      e.preventDefault();
      onCloseRef.current?.('escape');  // ref 통해 최신 콜백 호출
    }
  };
  // ...
}, [open]);  // onClose 의존성 제거
```

- 메인 effect 의존성은 `[open]` 만 — `open` 변화 시에만 setup/cleanup.
- Esc 핸들러는 `onCloseRef.current` 로 항상 최신 콜백 호출.
- 다른 닫기 경로(top-x, bottom-close, backdrop)는 이벤트 핸들러 안에서
  `onClose?.(...)` 직접 호출이라 매 렌더 최신 클로저 캡처 — 무영향.

### I-2 해결: 별도 backdrop 레이어 + pointer-events 분리

**문제 분석:**
- 기존: root `<div onClick={handleBackdropClick}>` 가 `flex flex-col` 인데 자식
  `<header>`, `<div flex-1>`, `<footer>` 가 root 전 면적을 덮음.
- `e.target === e.currentTarget` 조건이 만족되는 root 의 빈 픽셀이 사실상 0.
- backdrop 클릭이 동작하지 않음 → 4가지 닫기 중 하나(외부 탭) 사실상 비활성.

**수정 (BoothMinimapModal.jsx:110-130):**
```jsx
<div
  ref={ref ?? dialogRef}
  role="dialog"
  /* ... */
  className="fixed inset-0 z-50"  // flex flex-col 제거 → wrapper 만
>
  {/* Backdrop 레이어 — absolute inset-0, 클릭 시 onClose('backdrop') */}
  <div
    aria-hidden="true"
    onClick={() => onClose?.('backdrop')}
    className="absolute inset-0 bg-black/80"
    data-testid="modal-backdrop"
  />

  {/* 콘텐츠 컨테이너 — relative z-10, pointer-events-none 으로 빈 영역 통과 */}
  <div className="relative z-10 flex flex-col h-full pointer-events-none">
    <header className="... pointer-events-auto">...</header>
    <div className="flex-1 ... pointer-events-auto">...</div>
    <footer className="... pointer-events-auto">...</footer>
  </div>
</div>
```

- backdrop은 별도 absolute 레이어로 분리 → 명시적 클릭 타깃.
- 콘텐츠 컨테이너는 `pointer-events-none` → 빈 영역(콘텐츠 사이) 클릭은 backdrop 통과.
- 각 자식(header/main/footer)은 `pointer-events-auto` → 자체 클릭 정상 동작.
- `e.target === e.currentTarget` 조건 제거 — backdrop 레이어 onClick 은 항상 의미 명확.

## 결정 근거

### I-1: latest ref vs eslint-disable

리뷰 메시지의 옵션 2개 중 **latest ref 패턴** 채택:
- eslint-disable 은 향후 유지보수자에게 의도 불명확.
- latest ref 는 React 공식 권장 패턴 (Dan Abramov, React docs `useEffectEvent` RFC).
- effect 가 `open` 한 변화에만 반응한다는 의미가 명확.
- Esc 핸들러가 최신 onClose 를 호출한다는 회귀 테스트도 가능.

### I-2: 별도 backdrop 레이어 vs stopPropagation

리뷰 메시지의 권장(가장 단순) 채택 — **별도 absolute backdrop 레이어 + pointer-events 격리**.
- stopPropagation 패턴은 모든 자식 컴포넌트가 의식해야 함 (취약).
- absolute 레이어 + pointer-events 패턴은 구조적으로 격리 → 향후 자식 추가에 안전.
