### 3. GREEN — 3 atom 구현

#### Icon.jsx (얇은 래퍼)

판단 — IMPLEMENTATION_PLAN의 children-as-function 패턴(복잡)·아이콘 자체 prop(barrel 강제) 둘 다 피하고, *children 으로 lucide-react SVG 그대로 받는* 가장 단순한 형태로 채택.

사용 예:
```jsx
import { Map } from 'lucide-react';
import Icon from '@/components/atoms/Icon';

<Icon label="부스 약도 열기"><Map size={20} /></Icon>
<Icon decorative><ShoppingCart size={16} /></Icon>
```

- label 시 role=img + aria-label.
- decorative 시 aria-hidden=true (role / aria-label 모두 미부여).
- 사이즈는 호출자 SVG의 size prop (lucide-react 자체 prop)이 담당.

#### Spinner.jsx

```jsx
<span role="status" aria-label={label}
      className={`inline-block ${sizeCls} border-current border-r-transparent rounded-full animate-spin motion-reduce:animate-none ...`} />
```

- sm: `w-3 h-3 border` (12px·1px) — UI inline.
- md: `w-4 h-4 border-2` (16px·2px) — Button 내부 기본.
- lg: `w-6 h-6 border-2` (24px·2px) — 카드/풀스크린.
- `motion-reduce:animate-none` — `prefers-reduced-motion: reduce` 시 정지(DESIGN §9.5).

#### Divider.jsx

```jsx
const variantClasses = {
  solid: 'border-t border-divider',
  dashed: 'border-t border-dashed border-divider',
  stamp: 'border-t-2 border-dashed border-stamp-red opacity-70',
};
```

- label 없으면 `<hr>` 단순 출력(HTML 시맨틱 + 암시적 role=separator).
- label 있으면 `<div role="separator" aria-orientation="horizontal">` 안에 `[줄, 텍스트, 줄]` flex 패턴 — 양쪽 줄은 `aria-hidden="true"`.

### 4. Button atom 갱신 — DRY

기존 Button.jsx 내부의 inline `Spinner()` 함수를 제거하고 `import Spinner from './Spinner.jsx'` 추가.
사용처는 `<Spinner size="md" />`로 교체. 기존 Button 테스트(role=status + aria-label="로딩 중")는 시그니처 동일 → 변경 없이 통과.

### 5. 번들 위생 회귀 자동화 — §3.5 8조 핵심

`src/__tests__/bundle.test.js`에 두 번째 describe 추가:

```js
describe('번들 위생 — lucide-react named import 강제 (§3.5 8조)', () => {
  // src 트리 .js/.jsx 재귀 수집 (bundle.test.js 자기 자신 제외 — regex 문자열 자기 참조 매칭 회피).
  // 패턴 ①: import * as X from 'lucide-react'  (namespace barrel)
  // 패턴 ②: import X from 'lucide-react'      (default — named 중괄호 없는 형태)
  // 매칭되면 expect.not.toMatch 실패 → 자동 차단.
});
```

회귀 시나리오 — 누군가 `import * as Icons from 'lucide-react'` 추가하면 즉시 CI 빨강.

## 결정 노트

- **Icon children-as-function 패턴 채택 X** — IMPLEMENTATION_PLAN 가이드의 복잡 버전 제거하고 *children에 SVG 그대로 주입*하는 단순 형태로. 호출자가 lucide-react named import + size prop 직접 제어.
- **Icon이 size 매핑 안 함** — lucide-react 자체가 `size` prop을 받으므로 래퍼가 중복 매핑하면 혼란. 래퍼는 정렬 + aria 표준화만 담당.
- **Divider label 시 div + role=separator** — `<hr>`은 자식 노드를 가질 수 없는 void 요소이므로 label 있을 땐 div로 전환. WAI-ARIA `aria-orientation="horizontal"` 명시.
- **bundle.test.js 자기 참조 회피** — 정규식 문자열 자체가 `import * as X from 'lucide-react'` 패턴과 매칭되는 문제. 자기 자신을 collectSourceFiles 결과에서 filter로 제외.
