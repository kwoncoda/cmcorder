### 3. 부트스트랩 흐름 (`bootstrap.js`)

```js
export function bootstrapDatabase(db) {
  const tableExists = db.prepare("SELECT ... AND name='_migrations'").get();
  if (!tableExists) { runInitSql(db); return { initialized: true }; }

  const count = db.prepare('SELECT COUNT(*) AS c FROM _migrations').get().c;
  if (count === 0) { runInitSql(db); return { initialized: true }; }

  return { initialized: false };
}
```

- `_migrations` 테이블 미존재 → 신규 DB
- 테이블은 있지만 비어있음 → 이전 부팅 도중 실패한 케이스 → 재실행
- 행 있으면 skip

`runInitSql`은 `db.transaction(() => db.exec(sql))`로 단일 트랜잭션. 부분 적용으로 인한 깨진 스키마 방지.

### 4. 어드민 자동 시드 (`seedAdmin`)

```js
export function seedAdmin(db) {
  if (db.prepare('SELECT COUNT(*) AS c FROM admins').get().c > 0) return null;

  const envPin = process.env.DEFAULT_ADMIN_PIN;
  let pin, generated;
  if (envPin && /^\d{6}$/.test(envPin)) { pin = envPin; generated = false; }
  else { pin = String(crypto.randomInt(0, 1_000_000)).padStart(6, '0'); generated = true; }

  db.prepare('INSERT INTO admins (pin_hash) VALUES (?)').run(hashPin(pin));

  if (generated) console.log(`...\n[INIT] Generated admin PIN: ${pin}\n...`);
  return generated ? pin : null;
}
```

- env 형식 위반(6자리 숫자 아님) → 무시 + 랜덤 fallback (env 신뢰 X, 운영자 실수 방어)
- `crypto.randomInt(0, 1_000_000)` 균등 분포 → `padStart(6, '0')` 으로 `000000`도 안전
- stdout 출력은 `console.log` 1회 + pino logger 1회 (구조화 로그 + 운영자 강조)

### 5. PIN 해시 (`hashPin` / `verifyPin`)

```js
export function hashPin(pin) {
  return crypto.createHash('sha256').update(pin).digest('hex');
}

export function verifyPin(pin, hash) {
  if (!pin || !hash) return false;
  const computed = hashPin(pin);
  if (computed.length !== hash.length) return false;
  try { return crypto.timingSafeEqual(Buffer.from(computed, 'hex'), Buffer.from(hash, 'hex')); }
  catch { return false; }
}
```

- SHA-256 hex (Task 6.1 명세). DB_DRAFT §2.8 주석은 scrypt를 언급하지만, 본 Task 가이드는 SHA-256.
- 향후 운영 준비(Phase 7)에서 scrypt 마이그레이션 검토 가능 — 별도 ADR.
- `timingSafeEqual`은 길이 다른 buffer에서 throw → `verifyPin`이 false로 흡수.

### 6. 서버 통합 (`app.js` / `server.js`)

- `createApp({ db } = {})` — 테스트(supertest)는 db 없이 호출 가능, 운영은 `createApp({ db })` 주입
- `app.locals.db = db` — Phase 6 후속 라우트에서 `req.app.locals.db.prepare(...)` 사용 예정
- `server.js`는 `openDatabase → bootstrapDatabase → seedAdmin → app.listen` 순서
- graceful shutdown에 `db.close()` 추가 (catch로 감싸 무시) — Docker stop 시 WAL flush

## 결정 근거

1. **SHA-256 vs scrypt** — Task 6.1 가이드가 SHA-256 + timingSafeEqual을 명시. DB_DRAFT §2.8은 scrypt를 권장하지만 본 Task 범위 밖. Phase 7 운영 준비에서 재검토.
2. **메뉴 가격 SoT** — 가이드 SQL의 잘못된 가격(MED_KIT 20000 등)이 아닌 `src/constants/menus.js`의 SoT 가격(MED_KIT 21000 등)을 사용. `tests/unit/menus.test.js`가 클라이언트 SoT를 보호하므로 백엔드도 동일해야 회귀 X.
3. **business_state operating_date NOT NULL** — DB_DRAFT §2.13은 NULL 허용이지만, 본 Task 가이드 명세는 NOT NULL. 시드에서 `2026-05-20` 명시. G14 운영일과 일치.
4. **single-row enforcement** — `business_state.id PRIMARY KEY CHECK(id=1)`로 두 번째 행 INSERT 시도 차단 (DB_DRAFT §2.13 불변식과 일치).
5. **trans-action wrapping** — `db.transaction(fn)()` 사용 (better-sqlite3 API). 내부 throw 시 ROLLBACK 자동.
6. **env 검증 엄격** — `^\d{6}$` 형식 위반 시 env 무시 + 랜덤 fallback. 운영자가 `DEFAULT_ADMIN_PIN=admin` 같은 실수를 막음.
