## 한 일

1. **레거시 정리** — `node_modules/`, `public/{css,images,js}`, `data/` 삭제.
   네 디렉터리 모두 `.gitignore`에 있어 untracked 였으므로 git 영향 0. `.env`,
   `docs/`, `comorder*.zip`, `.git/`, `.claude/`, `.agents/`, `skills-lock.json`,
   `CLAUDE.md`, `README.md`는 손대지 않음.
2. **디렉터리 골격 생성** — `mkdir -p` + `touch` 한 줄로 23개 폴더 · 21개 `.gitkeep`
   배치. `find src server public tests -type d` 출력으로 검증.
3. **package.json 수동 작성** — `npm create vite` 대신 직접 작성. 기존 디렉터리에
   scaffold 도구가 충돌을 일으킬 위험 회피.
4. **`npm install`** — 171 packages, 21초. moderate 취약점 5건은 dev 의존성(jsdom 등)
   소속이라 보안 영향 적음. Task 0.4 테스트 인프라 단계에서 재점검 예정.
5. **RED 단계** — `src/__tests__/smoke.test.jsx` 먼저 작성.
   `npm test` → `Failed to resolve import "../App.jsx"` 에러로 1 failed.
   *로그 캡쳐 완료*.
6. **GREEN 단계** — `src/App.jsx` 최소 구현, `src/main.jsx` 진입점, `index.html`
   작성. `npm test` → `1 passed`. *로그 캡쳐 완료*.
7. **검증** — `npm run build` 성공 (dist 328 kB / gzip 100 kB). `npm run dev` →
   `http://localhost:5173/` HTTP 200, `/src/App.jsx` 변환 응답 확인.
8. **정리** — dev server 종료, `dist/` 삭제, `.gitignore`에 `dist/` 추가.
