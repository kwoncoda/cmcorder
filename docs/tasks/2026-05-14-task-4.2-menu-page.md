## 한 일

### 1. `useMenuData` hook
- `useApi` + `apiFetch(API.MENUS, { schema: MenuListSchema })` 캡슐화
- popular **파생** (별도 fetch X — 결정 E):
  - `recommended=true` 메뉴 → TOP 3 (학생회 토글)
  - 없으면 menus 첫 3개 fallback
- 반환: `{ menus, popular, isLoading, error, refetch }`

### 2. `RecommendedBanner` organism
- 빈 popular 시 `null` 반환 (영역 비움)
- 카피 **"🔥 학생회 추천 BEST"** — `text-ink` 본문 (형광 옐로 텍스트 X — AI 슬롭 #26)
- MenuCard 3개 `grid-cols-3` + `recommended=true` 도장 표시

### 3. `CategoryTabs` organism
- `role="tablist"` + `role="tab"` + `aria-selected` — 단순형 WAI-ARIA tab 패턴
- 활성 탭: `bg-accent text-card-ink` — **배경**만 형광, 텍스트는 #1B1F23 (AI 슬롭 #26 회피 — 텍스트가 아니라 배경)
- `min-h-[44px]` — 모바일 hitbox

### 4. `MenuList` organism
- `grid-cols-2` 모바일 기본 2열 (8개라 4행)
- `soldOut` / `recommended` 메뉴 필드를 MenuCard 에 그대로 위임 — 흐림 + SOLD OUT 도장 자동
- 빈 배열은 호출자(MenuPage)가 EmptyState 로 분기

### 5. `StickyCartBar` organism — §3.5 2조 핵심
- **`useCartStore(cartSelectors.totalQty)` + `useCartStore(cartSelectors.totalPrice)` 직접 구독**
- props drilling 없음 — 부모(MenuPage)는 onCheckout 콜백만 전달
- `totalQty === 0` 시 `null` 반환 (불필요 UI 가림 회피)
- `fixed bottom-0 z-40` — 모든 페이지 하단 떠 있음

### 6. `MenuPage` 페이지 (98줄, ≤120 OK)
- `useMenuData()` hook 호출 → 3분기 분기:
  - `isLoading` → `<LoadingState variant="page" minimumDelay={0}>` (wrapper `data-testid="menu-page"` 유지 — App 라우팅 회귀)
  - `error` → `<ErrorState variant="page" actionLabel="다시 시도" onAction={refetch}>`
  - 데이터 도착 → 합성 영역
- 합성 영역:
  - `<RecommendedBanner menus={popular} onAdd={addItem} />`
  - `<CategoryTabs ... value={category} onChange={setCategory} />`
  - `filteredMenus.length === 0 ? <EmptyState .../> : <MenuList .../>`
  - `<StickyCartBar onCheckout={() => navigate('/cart')} />`
- `addItem` 만 store 구독 (액션 — 안정적, 리렌더 유발 X)

### 7. 테스트 패턴 결정
- MenuFallback 이 메뉴 name 을 별도 span 으로 노출 → 카드 한 개당 본명 2회 등장
- 해결: `getByRole('heading', { name })` 패턴 — MenuCard h3 만 매칭 (MenuFallback span 무시)
- 페이지 줄 수 회귀: `node:fs.readFileSync(path.resolve(process.cwd(), '...'))` — `import.meta.url` 은 jsdom 환경에서 file: 스킴 보장 X
