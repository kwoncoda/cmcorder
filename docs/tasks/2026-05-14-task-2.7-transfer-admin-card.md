### 2. GREEN — 구현

#### TransferReportForm.jsx 핵심

- **모듈 최상위 BANK_OPTIONS** (§3.5 6조) — placeholder 1 + 6 은행 + 기타.
- **plain controlled state** (`useState`) — react-hook-form은 Phase 3 도입.
  `onSubmit({bank, customBank?, depositorName, useOtherName, otherName?, amount})`
  콜백 시그니처는 이후 react-hook-form 적용 시 동일.
- **touched state** — onBlur 시 에러 표시. 제출 시 모든 필드 강제 touched.
- **조건부 input**: `bank === '기타'` → customBank, `useOtherName` → otherName.
- **amount 정제**: `replace(/[^\d]/g, '')` — "18,000원" 자유 입력 허용.
- **검증**: 단순 (Phase 3 zod 교체 예정) — 빈값 + 양수만.
- **forwardRef** — 호출자가 form DOM 직접 접근 (focus 제어 등).
- **error prop → `<p role="alert" data-testid="form-error">`** — 서버 에러 알림.

#### AdminCardColumn.jsx 핵심 (§3.5 7조 메모이즈)

- **OrderCard 모듈 최상위 named export** — AdminCardColumn 안에 정의 X.
  안에 정의하면 부모 리렌더마다 새 함수 reference → memo 무용지물.
- **`memo(function OrderCard(...))`** — props shallow compare.
  동일 `order` reference + 동일 `tick` 시 리렌더 X.
- **`useMemo([order.transferred_at, tick])`로 elapsed_minutes** — tick은
  부모(KanbanBoard)가 1분 단위 발행. transferred_at 안 바뀌면 캐시.
- **`elapsedTone(min)`**: `>=10 → border-danger`, `>=5 → border-warning`,
  `else → border-divider`.
- **key={order.id}** — index 금지 (§3.5 7조).
- **시맨틱 `<button>`** — 키보드 Enter·Space 기본 + focus-visible 형광 옐로.
- **카드 톤**: `bg-card-bg + text-card-ink` — 본문(`bg-elevated`)과 구분 (AI 슬롭 #26).

### 테스트 보정 사유

1. **Label required → `<span>*</span>`** 추가 텍스트 노드로 accessible name이 "은행 *"이 됨.
   `getByLabelText('은행')` → `getByLabelText(/^은행/)` 정규식으로 교체.
2. **placeholder option `<option>은행을 선택하세요</option>` + error `<p>은행을 선택하세요</p>`**
   동시 존재 → `getByText` 두 개 매칭 에러. `getAllByRole('alert')` + textContent 매칭으로 교체.
3. **`<Button loading>`은 Spinner로 children 교체**, accessible name이 "로딩 중".
   `getByRole('button', { name: /이체 신고 제출/ })` → `name: /로딩 중/`로 교체.
