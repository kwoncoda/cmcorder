## 한 일

### StatusPage 핵심 결정

1. **§3.5 5조 — 부수효과는 이벤트 핸들러로**:
   - `handleStatusChange(prev, next)` 안에서 `prev !== 'READY' && next === 'READY'` 일 때만 `navigator.vibrate([200,100,200])` + `setPulse(true)`.
   - useEffect deps에 status 두지 X — useOrderStream 자체가 prev=null 시 콜백 호출 X (회귀: 새로고침 후 status=READY 직진입 시 진동 0회).
2. **SSE snapshot 우선** — `stream.snapshot ?? initialQuery.data` 패턴으로 실시간 갱신은 SSE가, 초기/새로고침은 useApi가 담당.
3. **3분기 처리**: Loading / Error(5xx) / 404 redirect (useEffect 안에서 — 렌더 도중 navigate 회피).
4. **OrderTimeline 미니뷰** — `history` prop으로 `created_at`·`paid_at` 등 시각 전달. ADR-010 보존 — 추정·예측 X, "—" fallback.
5. **aria-live polite** — `<div role="status" aria-live="polite">` 안에 STATE_LABEL 카피. 상태 변경 시 SR이 자동 announce.
6. **SSE 끊김 안내** — `stream.isConnected === false` 시 `<p role="alert" data-testid="sse-disconnected">` 노출. 브라우저 자동 재연결.

### MapPage 결정

1. **합성만** — BoothMinimapModal organism이 이미 모달 기능 전부 (4 닫기 방식, focus trap, scroll lock). 페이지는 라우팅 어댑터.
2. **myTableNo 결정** — `?order_id=N` 쿼리에서 추출. 미지정 시 undefined (그리드 fallback에서 강조 X).
3. **닫기 = navigate(-1)** — 닫기 시 이전 페이지로 history back. MemoryRouter 테스트로 회귀 보호.

### ClosedPage 결정

1. **합성만** — ClosedScreen organism이 4 reason 모두 처리 + 운영 일정 카드.
2. **쿼리 → props** — `?reason=` 와 `?date=` 를 그대로 전달. 기본값 `'before-open'` / `'2026-05-20'`.
3. **새로고침 CTA** — `window.location.reload()` — G13 reactive 단일 진입점.

### ErrorPage 결정

1. **MascotState canceled** — 😢 fallback 이모지. useFallback=true (D-3 자산 미수령).
2. **404/500 분기** — ERROR_MESSAGES 모듈 최상위 SoT. 알 수 없는 code → 404 fallback.
3. **role="alert"** — 즉시 SR announce. CTA는 `<Link to="/menu">` 로 라우팅.
