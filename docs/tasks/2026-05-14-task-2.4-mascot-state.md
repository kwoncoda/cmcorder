**img/이모지 분기 + onError 안전망:**

```jsx
const [imageFailed, setImageFailed] = useState(false);

```

- `useFallback=true` (기본) — 처음부터 이모지 (자산 미수령 단계, public/mascot/ 비어있음).
- `useFallback=false` — img 시도. `onError` 시 `setImageFailed(true)` → 이모지 안전망 복귀.

**cross-fade — 단순한 매 마운트 모션:**

```css

```

- 모든 마운트 마다 발화. 부모가 `<MascotState key={state}/>` 패턴으로 unmount/remount 시
  state 전환이 자연스러운 cross-fade. 추가 React 상태 불필요.

**cooking idle 흔들:**

```css
@keyframes mascot-cooking-idle {
```

- `state === 'cooking'` 일 때만 클래스 합성. 다른 state 에는 없음.

**reduced motion 정적화 — components.css 기존 media query 확장:**

```css

  }
}
```

## 결정

- **결정 c (자산 D-3 미수령) 충실 반영:** `useFallback=true` 기본 → 마스코트 png 자산 없이도 컴포넌트 정상 동작.
  자산 수령 후 부모 호출부에서 `useFallback={false}` 전환만 하면 됨 (혹은 prop 안 넘기고 import 시점에 STATE_CONFIG 의 src 수정도 가능).
- **state별 다양한 fallback 이모지:** 명세 회귀(`default = 🪖`) 충족 위에 시각적 구분(🏃/🔥/🎉/😢) 추가 — 운영 중 자산 누락 시에도 UX 자연스러움.
- **cross-fade 패턴:** 컴포넌트 내부 상태 없이 매 마운트마다 `.mascot-fade` 발화. 부모가 `key={state}`
  주는 표준 React 패턴으로 unmount/remount 트리거 → 자연스러운 state 전환.
