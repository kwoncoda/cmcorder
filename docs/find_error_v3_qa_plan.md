# find_error_v3 검수 지시서

## 검수 목적

이번 v3 검수의 목적은 실제 축제 운영에서 쿠폰 중복 사용을 학번 기준으로 막고, 관리자가 주문/메뉴/시스템 운영 기록을 확인할 수 있으며, 어드민 화면이 `docs/design-bundle` 기준에 더 가깝게 개선되었는지 확인하는 것이다.

특히 다음 항목을 중점 확인한다.

- 같은 학번의 쿠폰 중복 사용이 이름 변경으로 우회되지 않는가
- 쿠폰 중복 차단이 일반 주문 가능 여부를 막지 않는가
- 관리자 내역에서 전체/주문/메뉴/시스템 필터가 실제 운영 로그를 구분하는가
- 어드민 메뉴 페이지에 메뉴 효과가 정확히 표시되는가
- 이체확인 탭 제거 후에도 대시보드에서 이체 확인 처리가 가능한가
- 장사 시작 전/후 관리자 대시보드 UI가 현장 운영에 적합한가
- 관리자 화면의 불필요한 이모지가 제거되었는가

## 검수 전제

- 기존 운영 DB는 없으며 DB 초기화 후 검수한다.
- 자동 백업 로그는 v3 시스템 로그 범위에 포함한다.
- 관리자 로그인 로그는 v3 시스템 로그 범위에 포함한다.
- 이체확인 라우트/API/컴포넌트는 유지하고 nav에서만 제거한다.
- 메뉴 효과는 프론트 정적 매핑 기준으로 검수한다.

## 검수 시나리오

### 시나리오 1. 쿠폰 학번 기준 중복 방지

- 이름 A + 학번 `202637123`으로 쿠폰 사용
- 이름 B + 같은 학번 `202637123`으로 쿠폰 재사용 시도
- 쿠폰 재사용이 막히는지 확인
- 일반 주문은 가능한지 확인
- 에러 문구 `이미 쿠폰을 사용한 학번이에요.` 확인

### 시나리오 2. 쿠폰 일반 주문 분리

- 이미 쿠폰을 사용한 학번으로 쿠폰 없이 주문 가능
- 쿠폰 중복 때문에 주문 자체가 막히지 않는지 확인
- `coupon.used=false` 또는 쿠폰 미선택 요청이 정상 주문으로 처리되는지 확인
- 정산 금액에 쿠폰 할인이 잘못 적용되지 않는지 확인

### 시나리오 3. 관리자 내역 필터

- 내역 탭에 전체 / 주문 / 메뉴 / 시스템 필터가 보이는지 확인
- 전체에서 모든 로그가 보이는지 확인
- 주문 필터에서 주문 관련 로그만 보이는지 확인
- 메뉴 필터에서 메뉴 변경 로그만 보이는지 확인
- 시스템 필터에서 장사 시작/로그인/시스템 로그가 보이는지 확인
- 필터 전환 후 빈 상태 문구가 깨지지 않는지 확인

### 시나리오 4. 주문 로그 기록

- 주문 생성 시 주문 로그 기록
- 이체 완료 요청 시 주문 로그 기록
- 관리자 상태 변경 시 주문 로그 기록
- 주문 로그에 시간, 주문번호, 액션, 처리 주체가 표시되는지 확인
- 새로고침 후에도 로그가 유지되는지 확인

### 시나리오 5. 메뉴 로그 기록

- 메뉴 품절 처리 시 메뉴 로그 기록
- 품절 해제 시 메뉴 로그 기록
- 가격 변경 기능이 있다면 가격 변경 로그 기록
- 가격 변경 기능이 없다면 확인 필요로 기록
- 메뉴 로그에 메뉴명, 변경 전/후 값, 처리 주체가 표시되는지 확인

### 시나리오 6. 시스템 로그 기록

- 장사 시작 시 시스템 로그 기록
- 관리자 로그인 시 시스템 로그 기록
- 자동 백업 성공 시 시스템 로그 기록
- 시스템 시작 로그는 구현 범위에 포함했는지 확인
- 장사 시작을 이미 OPEN 상태에서 다시 눌렀을 때 중복 로그가 생기는지 확인

### 시나리오 7. 어드민 메뉴 효과 표시

- 후라이드: 회복량 +10
- 양념: 회복량 +75
- 뿌링클: 회복량 +100
- 감자튀김: 부활
- 뿌링감자튀김: 소생
- 칠리스: 부스트 +100%
- 콜라: 부스트 +60%
- 사이다: 부스트 +40%
- 어드민 메뉴 페이지에서 위 효과가 정확히 보이는지 확인
- 품절/가격 변경/추천 토글 후에도 효과 표시가 사라지지 않는지 확인

### 시나리오 8. 어드민 이체확인 탭 제거

- 어드민 nav에 이체확인 탭이 보이지 않는지 확인
- `/admin/transfers` 라우트와 `GET /admin/api/transfers`는 보존되는지 확인
- 대시보드에서 이체 확인 요청 처리는 계속 가능한지 확인
- `TRANSFER_REPORTED` 주문 카드에서 확인/보류 액션이 동작하는지 확인
- `HOLD` 주문 카드에서 이체 확인/취소 액션이 동작하는지 확인
- `/admin/transfers` 직접 접근 시 기존 이체확인 페이지가 유지되는지 확인

### 시나리오 9. 문구 변경

- 쿠폰 문구가 `컴모융 학생 1,000원 할인`으로 보이는지
- `컴모융(****38***) 학생만 쿠폰 사용이 가능해요` 문구가 보이지 않는지
- 현재 구현에 있던 `컴모융(****37***) 학생만 쿠폰 사용이 가능해요.` 문구도 보이지 않는지
- 어드민 화면의 `admin1` 표시가 `어드민`으로 보이는지
- 내역의 actor 표시도 `admin1` 또는 `admin` 대신 `어드민`으로 보이는지 확인

### 시나리오 10. 장사 시작 UI

- `docs/design-bundle`처럼 장사 시작 네모 칸 안 UI/UX가 반영됐는지
- 장사 시작 전에도 아래 6개 상태 박스가 보이는지
- 장사 시작 버튼이 너무 평범하거나 어색하지 않은지 확인
- 장사 시작 버튼 클릭 시 영업 상태가 OPEN으로 전환되는지 확인
- 장사 시작 실패 시 에러 안내가 카드 구조를 깨지 않는지 확인

### 시나리오 11. 어드민 이모지 제거

- 어드민 페이지의 불필요한 이모지가 제거되었는지
- 영업중/open 옆 초록색 동그라미는 유지되는지 확인
- 쿠폰, 정산, 메뉴, 내역, 대시보드 제목에 이모지가 남아있지 않은지 확인
- 주문 상태 칩이 관리자 화면에서 이모지 없이도 상태를 구분할 수 있는지 확인
- 고객 화면의 이모지 제거가 이번 범위가 아니라면 고객 화면이 의도치 않게 바뀌지 않았는지 확인

### 시나리오 12. 대시보드 박스 디자인

- 6개 상태 박스가 `docs/design-bundle`처럼 꾸며졌는지
- 내부 주문 카드가 `docs/design-bundle` 스타일과 유사한지
- 드래그 바/스크롤 UI가 이상하지 않은지
- 취소 버튼이 너무 튀는 빨간색으로 보이지 않는지
- 위험 액션임은 여전히 인지 가능한지 확인
- 주문 카드가 많을 때 컬럼 내부 스크롤이 자연스러운지 확인
- 모바일 화면에서 6개 박스가 겹치거나 텍스트가 넘치지 않는지 확인

## 자동 테스트 제안

가능하면 다음 테스트를 추가한다.

- 같은 `student_id` + 다른 이름 쿠폰 중복 차단 테스트
- 쿠폰 중복이어도 일반 주문 가능 테스트
- `used_coupons.student_id` unique 또는 equivalent 검증 테스트
- 기존 `student_id + name` 중복 허용 회귀 방지 테스트
- 내역 필터 전체/주문/메뉴/시스템 테스트
- 주문 이벤트 로그 테스트
- 메뉴 이벤트 로그 테스트
- 시스템 이벤트 로그 테스트
- 장사 시작 시스템 로그 테스트
- 관리자 로그인 시스템 로그 테스트
- 자동 백업 로그 테스트
- 관리자 메뉴 효과 표시 테스트
- 이체확인 탭 미노출 테스트
- 대시보드 이체 확인 액션 유지 테스트
- `admin1` → `어드민` 표시 테스트
- 쿠폰 문구 변경 테스트
- 쿠폰 대상 패턴 안내 문구 삭제 테스트
- 어드민 이모지 미노출 테스트
- 장사 시작 전 6컬럼 유지 테스트
- 장사 시작 버튼이 카드 내부에 있는지 확인하는 UI 테스트
- 취소 버튼 스타일 회귀 테스트
- 모바일 관리자 대시보드 레이아웃 테스트

## 수동 검수 체크리스트

- 브라우저에서 관리자 로그인 후 nav 항목이 `본부 / 메뉴 / 내역 / 정산 / 쿠폰`만 보이는지 확인
- 관리자 대시보드에서 CLOSED 상태를 만들고 장사 시작 카드와 6개 컬럼을 동시에 확인
- 메뉴 페이지에서 품절/가격 변경 후 내역의 메뉴 필터 확인
- 주문을 생성하고 이체 완료 요청, 이체 확인, 조리 시작, 조리 완료, 전달 완료를 순서대로 진행하며 주문 로그 확인
- 같은 학번으로 이름만 바꿔 쿠폰을 다시 시도하고 차단 문구 확인
- 같은 학번으로 쿠폰 없이 다시 주문해 성공 확인
- 정산 탭의 쿠폰 건수와 쿠폰 사용 내역 건수 확인
- 모바일 폭에서 관리자 nav, 대시보드 컬럼, 주문 카드 액션 버튼 확인

## 최종 통과 기준

- 같은 학번으로 쿠폰을 두 번 사용할 수 없음
- 이름을 바꿔도 쿠폰 중복 사용이 차단됨
- 쿠폰 중복이어도 일반 주문은 가능
- 관리자 내역에서 전체/주문/메뉴/시스템 필터가 동작
- 주문/메뉴/시스템 주요 이벤트가 새로고침 후에도 유지됨
- 메뉴 효과가 정확히 표시됨
- 이체확인 탭이 제거됨
- 대시보드에서 이체 확인 처리는 계속 가능함
- 쿠폰/어드민 문구가 정리됨
- 어드민 화면 불필요 이모지가 제거됨
- 영업중/open 옆 초록색 동그라미는 유지됨
- 장사 시작 UI와 대시보드 카드 UI가 `docs/design-bundle` 기준에 가까워짐
- 취소/보류 버튼이 과도한 빨강 배경으로 보이지 않으면서 위험 액션임을 알 수 있음
- test/lint/build 통과
- 모바일 화면에서 관리자 UI가 깨지지 않음
