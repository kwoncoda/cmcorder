## 1. OPEN/영업 중 초록 dot 복원

- 번호: 1
- 현재 문제
  - find_error_v3에서 어드민 이모지 제거 작업을 하면서 `🟢` 표시가 CSS dot으로 대체되었다.
  - 사용자 요청 기준으로 오른쪽 `영업 중(OPEN)` 상태 표시 옆 초록색 동그라미가 원본처럼 보이지 않는 문제가 있다.
- design_bundle 기준 화면/파일/컴포넌트
  - `docs/design-bundle/screens-admin.jsx`
    - `AdminApp` 상단 `.biz-badge.open`: 초록 원형 표시 + `OPEN`.
    - `AdminDashboardBody`의 `.open-status`: `.open-dot` 안에 pulse와 초록 dot이 함께 보인다.
  - `docs/design-bundle/app.css`
    - `.biz-badge.open`
    - `.open-status`
    - `.open-status .open-dot`
    - `.open-status .open-dot .pulse`
    - `@keyframes open-pulse`
- 현재 구현 파일/컴포넌트/CSS
  - `src/components/layouts/AdminLayout.jsx`
    - `.biz-badge` 안에 `<span className="biz-dot is-open" />OPEN`.
  - `src/components/organisms/BusinessStateBadge.jsx`
    - `영업 중 (OPEN)` 앞에 `.biz-dot.is-open`.
  - `src/pages/admin/DashboardPage.jsx`
    - `.open-status`의 `.open-dot` 안에는 `<span className="pulse" />`만 있고 실제 dot은 CSS `::before`에 의존한다.
  - `src/styles/components.css`
    - `.biz-dot`
    - `.open-status .open-dot::before`
    - `.open-status .open-dot .pulse`
- design_bundle과 현재 구현의 차이점
  - 원본은 상태 배지에서 초록 dot이 명시적으로 보이는 시각 요소다.
  - 현재 구현은 find_error_v3 정책에 따라 이모지를 제거하고 CSS dot으로 치환했으며, 일부 위치에서는 dot이 pseudo-element에 숨어 있어 렌더링/정렬/여백 회귀가 발생하기 쉽다.
  - `BusinessStateBadge`의 pill 형태는 원본 `.biz-badge.open`보다 큰 버튼형 배지에 가깝다.
- 수정 방향
  - `AdminLayout.jsx`의 우측 `.biz-badge.open`은 `design_bundle`처럼 `OPEN` 바로 왼쪽에 확실한 초록 원이 보이도록 복원한다.
  - `DashboardPage.jsx`의 `.open-status`도 원본처럼 `.open-dot` 영역에 pulse와 실제 초록 dot이 동시에 보이게 한다.
  - dot 크기/여백은 원본 감각에 맞춰 8~10px, `gap: 6px`, `var(--color-success)` 계열을 유지한다.
  - `BusinessStateBadge.jsx`는 `영업 중 (OPEN)` 문구 앞 dot을 유지하되 원본 `.biz-badge.open`과 충돌하지 않도록 크기와 색감을 맞춘다.
  - 이 작업은 이모지 복원 여부보다 "초록 원형 표시가 확실히 보이는 것"이 핵심이다.
- 절대 바꾸면 안 되는 기능 로직
  - `businessState` store의 상태 계산.
  - `/admin/api/business-state` 호출.
  - `OPEN/CLOSED` 전이 로직.
  - 정산 마감 시 `CLOSED` 전환 정책.
- 테스트/수동 QA 포인트
  - 어드민 상단 우측 `OPEN` 옆에 초록 dot이 보이는지.
  - 본부 대시보드 `.open-status`의 `영업 중` 왼쪽에 초록 dot과 pulse가 동시에 보이는지.
  - `CLOSED` 상태에서는 빨간 계열 상태 표시가 깨지지 않는지.
  - 모바일/태블릿 폭에서 dot이 텍스트와 겹치거나 사라지지 않는지.
- 심각도: P1

## 3. 어드민 메뉴 가격 설정 저장/취소 이모지 복원

- 번호: 3
- 현재 문제
  - find_error_v3에서 어드민 이모지 제거 정책으로 가격 편집 저장/취소 버튼의 `✓`/`✕`가 텍스트 `저장`/`취소`로 바뀌었다.
  - 현재 가격 편집 UI는 원본보다 폭이 넓고, 가격 셀 안에서 작은 아이콘 버튼처럼 보이지 않는다.
- design_bundle 기준 화면/파일/컴포넌트
  - `docs/design-bundle/screens-admin.jsx`
    - `AdminMenus`
    - `.price-cell`
    - `.bump-btn`
    - `.price-display`
    - `.saved-tick`
  - `docs/design-bundle/app.css`
    - `.price-cell`
    - `.bump-btn`
    - `.price-display`
    - `.saved-tick`
    - `.price-edit`
  - 이전 복원 근거
    - `docs/tasks/2026-05-18-find_error_v3.md`에 `✓→저장`, `✕→취소` 변경 기록이 있다.
- 현재 구현 파일/컴포넌트/CSS
  - `src/pages/admin/MenuAdminPage.jsx`
    - `save-price-*` 버튼 텍스트가 `저장`.
    - `cancel-price-*` 버튼 텍스트가 `취소`.
  - `src/styles/components.css`
    - `.bump-btn`은 원본의 작은 아이콘 버튼 스타일에 가깝지만, 현재 텍스트 버튼이 들어가면서 원본 비율과 달라진다.
- design_bundle과 현재 구현의 차이점
  - 원본 가격 셀은 작은 버튼, 숫자, 저장 tick 같은 compact control 느낌이다.
  - 현재 저장/취소가 한글 텍스트라 셀 폭과 정렬이 달라지고 원본의 작은 control 감각이 깨진다.
- 수정 방향
  - 가격 편집 중 저장/취소 버튼은 원래처럼 `✓`/`✕` 계열의 작은 버튼으로 복원한다.
  - 접근성은 `aria-label="저장"`, `aria-label="취소"`로 유지한다.
  - `.bump-btn`의 28x28 크기, 투명 배경, hover 시 accent 톤은 원본 CSS를 유지한다.
  - 가격 변경 API 호출과 optimistic update는 건드리지 않는다.
- 절대 바꾸면 안 되는 기능 로직
  - `PATCH/POST /admin/api/menus/:id` 호출.
  - `base_price` 변경 payload.
  - 품절/추천 토글 기능.
  - 가격 변경 로그 정책.
  - 서버 가격 계산 정책(ADR-020 Pattern B).
- 테스트/수동 QA 포인트
  - 가격 편집 진입 시 input 옆 저장/취소가 작은 `✓`/`✕` 아이콘 버튼으로 보이는지.
  - Enter 저장, Escape 취소 동작이 유지되는지.
  - 저장/취소 버튼의 aria-label이 남아 있는지.
  - 가격 저장 후 실제 메뉴 가격이 반영되는지.
- 심각도: P3
